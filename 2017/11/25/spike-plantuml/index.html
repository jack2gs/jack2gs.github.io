PlantUML实践

Posted on 2017-11-25 | Post modified 2017-11-25 | In PlantUML , UML

本文是博主的PlantUML学习，实践与总结。希望对你有帮助。

1 时序图（Sequence Diagram）

1.1 简单示例(Basic examples)

1.1.1 要点

  • -> 表示实线箭头
  • --> 表示虚线箭头

1.1.2 实践

1.1.3 总结

这里需要注意的是 @startuml 之前和 @enduml 之后不可以有空行, 否则会出错。

1.2 声明参与者(Declaring participant)

1.2.1 要点

  • 关键字 participant 用于改变参与者的顺序。此外还有其他的关键字可以声明参与者：

    名称 用途
    actor 参与者
    boundary 边界
    control 控制
    entity 实体
    database 数据库
  • 关键字 as 用于重命名参与者
  • 可以自定义各种参与者的背景颜色

1.2.2 总结

  • 自定义参与者，并更改显示顺序 participant, actor, boundary, control, entity, database 和 =collections=用于声明不同的参与者
  • 使用 as 重命名参与者
  • 自定义颜色 可以使用 RGB 和颜色英文名来指定颜色。
  • 注释
    • 使用 ' 定义单行注释
    • 使用 /'...'/ 定义多行注释

1.3 在参与者中使用非字母符号（Use non-letters in participants）

1.3.1 要点

PlantUML允许使用非字母符号，比较常用的，例如换行符 \n.

1.3.2 实践

1.3.3 总结

注意这两种写法是等效的： 

"Bob()" -> "This is very\nlong" as Long
' is same with
"Bob()" -> Long As "This is very\nlong"

1.4 给自己发消息（Message to self)

1.4.1 要点

  • 参与者可以给自己发消息
  • 消息文字可以用 \n 来换行，前面提到了在参与者中使用非字母字符，这里与前者类似。

1.4.2 实践

1.4.3 总结

最好在操作符两边加上空格。

1.5 修改箭头样式(Change arrow style)

1.5.1 要点

修改箭头的方式有以下几种：

  • 表示一条的消息： 末尾加x
  • 让箭头只有上半部分或者下半部分：将 < 和 > 替换成 \ 和 /
  • 细箭头： 将箭头标记写两次（如 >> 或者 //)
  • 虚线箭头: 用 -- 替代 -
  • 箭头末尾加圈： ->o
  • 双向箭头： <->

1.5.2 实践

1.5.3 总结

不同的箭头样式可以表示不同的消息类型。

1.6 修改箭头颜色（Change arrow color）

1.6.1 要点

可以使用颜色名字或者RGB值表示颜色

1.6.2 实践

1.6.3 总结

有时候在想，这些花哨的东西有没有必要。。。

1.7 对消息序列编号（Message sequence numbering）

1.7.1 要点

  • 关键字 autonumber 用于自动对消息编号。

  • autonumber [<start>] [<increment>] ["<format>"] 可以同时指定编号的初始值，增量和格式。

    参数 类型 含义 备注
    start 整型 编号起始值  
    increment 整型 编号增量  
    format 字符串 编号格式 1.引号必须要有；2.格式语法参考 Java 的 =DecimalFormat=类实现的
  • autonumber stop 暂停自动编号
  • autonumber resume [<increment>] ["<format>"] 恢复编号 格式参考上述表格.

1.7.2 实践

1.7.3 总结

编号可以让消息的时序更清楚简明。

1.8 分割示意图（Splitting diagrams）

1.8.1 要点

关键字 newpage 用于把一张图分割成多张。

1.8.2 实践

1.8.3 总结

当时序图太过复杂时，可以分页。

1.9 组合消息

1.9.1 要点

  • 可以通过以下关键字组合消息
    • alt/else
    • opt
    • loop
    • par
    • break
    • critical
    • group, 后面紧跟着消息内容
  • 这些关键字，可以在header后添加需要显示的文字（group除外）
  • 关键词 end 用来结束分组
  • 分组可以嵌套使用

1.9.2 实践

1.9.3 总结

组合很强大， 以前用的比较少。使用plantuml还挺方便的。

1.10 给消息添加注释

1.10.1 要点

  • 给消息添加注释 这种类型的注释需要紧跟消息即可： note <left|right>

  • 多行注释 当需要多行注释时，需要使用 end note 语法即可。 

    A -> B
note <left|right> [<color>]
...
end note

  • 给 participant 添加注释 

    A -> B
note <left|right> of <participant> [<color>]
...
end note

1.10.2 实践

1.10.3 总结

注释很有必要，这也是一种反馈。
